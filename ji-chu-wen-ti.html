什么是左值引用和右值引用

可取地址的是左值，不可取地址的是右值。
左值引用是借用，右值引用是接管

C++move的作用和原理？

核心作用：将对象的资源所有权从一个对象“转移”到另一个对象，从而避免不必要的深拷贝，提高程序性能。
原理：仅仅是类型转换工具，不会实际移动数据，真正的移动逻辑由类的 移动构造函数 或 移动赋值运算符 决定。

static关键字的作用？什么场景下用？

static用于控制变量或函数的作用域和生命周期。

  1. 修饰局部变量（函数内部）
  • 在函数作用域内可见，但只会初始化一次，生命周期贯穿整个程序。
  • 使用场景：主要用于记录函数调用次数，实现懒加载
    ```
    #include
    using namespace std;

void func() {
static int count = 0; // 只在第一次调用func时初始化
cout << “Count is: “ << count << endl;
count++;
}

int main() {
for(int i = 0; i < 5; i++) {
func(); // 每次调用都会显示增加的count值
}
return 0;
}

2. 修饰全部变量（文件作用域）
* 限制变量/函数的作用域只在当前源文件中，避免全局命名冲突
* 使用场景：模块化开发中隐藏模块实现细节

// file1.cpp
static int count = 10; // count变量只能在file1.cpp中访问
static void func() { // func函数只能在file1.cpp中访问
cout << “Function in file1” << endl;
}
// file2.cpp
extern int count; // 这里会导致编译错误，因为count是static的，不能在file2.cpp中访问

void anotherFunc() {
func(); // 这里也会导致编译错误，因为func是static的，不能在file2.cpp中访问
}


3. 修饰类的成员变量
* 被修饰的成员变量属于类而不是变量，所有对象共享一份数据
* 使用场景：计数器、全局配置、缓存等（想要类的所有对象共享某个变量）
4. 修饰类的成员函数
* 被修饰的成员函数不依赖类对象就能调用
* 使用场景：访问静态变量、工具函数、工厂函数等（想要在没有某个类实例的情况下就访问某个函数）

#include
using namespace std;

class MyClass {
public:
static int staticValue; // 静态成员变量
static void staticFunction() { // 静态成员函数
cout << “Static function called” << endl;
}
};

int MyClass::staticValue = 10; // 静态成员变量的初始化

int main() {
MyClass::staticFunction(); // 调用静态成员函数
cout << MyClass::staticValue << endl; // 访问静态成员变量
return 0;
}


## const关键字的作用？谈谈你是怎么理解const的？
1. 作用：const最主要的作用就是声明一个变量为常量，这个变量在初始化之后就不被修改，不仅可以用于普通常量，还可以用于指针、引用、成员函数、成员变量等
* 定义普通常量：修饰基本数据类型的变量时，表示常量，对应的值不能被修改。  
`const int MAX_SIZE = 100; // MAX_SIZE是一个常量，其值不能被修改
`
* 修饰指针：分多钟情况，比如指针本身是常量，指针指向的数据是常量，或者指针本身及其指向的数据都是常量。
    * 指向常量的指针：不能通过该指针修改该常量的值  
    `const int* ptr = &x; // ptr是一个指向常量的指针，不能通过ptr修改x的值`
    * 指针常量：指针指向的地址不能被修改，但可以通过该指针修改其所指向的值（如果指向的值不是常量）  
    `int* const ptr = &x; // ptr是一个常量，ptr的值（地址）不能被修改，但x的值可以被修改`
    * 指向常量的常量指针：指针本身是常量，且指针指向的内容也是常量。
    `const int* const ptr = &x; // ptr的值和ptr指向的值都不能被修改`
* 修饰引用：const修饰引用的时候，一般用作函数参数，表示函数修改传递的参数值
`void func(const int& a) { // a是一个对常量的引用，不能通过a修改其值  
    // ...  
}
`
* 修饰类成员函数：表示函数不会修改类的任何成员变量，除非这些变量被声明为mutable
`class MyClass {  
public:  
    void myFunc() const { // myFunc是一个const成员函数，它不会修改类的任何成员变量  
        // ...  
    }  
};
`
* 修饰类成员变量：表示生命周期内不可改动此值
`class MyClass {  
public:  
    const int a = 5;
};
`

## define和const的区别？
1. 区别：
* #define是一个预处理指令，用于定义宏，只在预处理阶段进行文本替换，并且不进行类型检查，出现问题之后调试比较困难。
* const是一个关键字，用于定义常量，在编译时确定类型和值，并且具有类型安全的特点（定义常量优先使用const）
1. 扩展知识
* 作用机制
    * `#definde x 10`只是在预处理阶段，将所有出现的x替换为10,不进行类型检查，因此#define可以用于定义常量和函数
    * `const int x = 10`在编译阶段真实的定义了一个类型为int的常量x并赋值为10,保留了类型信息，也能进行常规的语法和语义检测
* 类型安全
    * `#define`没有类型，只是进行文本替换
    * `const`有完整的类型信息
* 作用域
    * `#define`定义的宏在整个源文件中有效
    * `const`只在指定的作用域内有效

## inline关键字的作用？有什么优缺点？
inline的作用是建议编译器将函数条用替换为函数体，以减少函数调用的开销，和宏比较类似。使用inline函数的目的一定是希望可以提高程序的运行效率，特别是那些频繁调用的小函数。
1. 基础用法
在定义的函数前加inline
2. 底层原理
正常在调用函数前要进行参数压栈、跳转、返回值处理，会有一定的开销，而使用内联函数后，编译器在编译阶段把函数调用的地方用函数体内容展开替换掉，跟宏替换类似。（不同的是inline有类型检查，更加安全）
1. 应用场景
* 小函数写在类内或者头文件中自动视为inline，或者需要避免重复定义时
* 需要对频繁调用的小函数进行内联，避免压栈时出栈开销。（提升性能）
2. 内联函数和#define的区别
* 宏定义实在预处理阶段进行文本替换，而内联函数在编译阶段展开。
* 内联函数有作用域控制和类型安全，宏没有这种特性。
* 内联函数调试起来更加方便。
3. 注意事项
* 函数体过于复杂或者递归调用的时候不要用内联函数

## sizeof和strlen的区别？
sizeof用于计算类型大小，strlen用于计算字符串长度。
1. 区别
* sizeof是一个编译时运算符，用于获取一个类型或对象的大小（以字节为单位），sizeof在编译时计算结果，不涉及实际内容。
* strlen是一个库函数，只能用于计算C风格字符串的长度（不包括终止运算符‘\0’，且不能用于string类型，身string类型有自己的成员函数size()和length()）。strlen在于形式计算结果，因为他需要遍历字符串内容。

2. 扩展知识
* sizeof的应用
    * 用于计算基础类型的大小
    * 用于计算结构体或类的内存占用
    * 对于静态数组，可以获取整个数组的内存大小
    * 对于指针，sizeof返回的是指针本身的大小而不是指针 指向内存区域的大小。
    ```
    int *p = new int[10];
    std::cout << sizeof(p); // 这会返回指针的大小，通常是4或8个字节，具体取决于系统架构。
    ```
* strlen的应用
    * 对于一些特定的字符数组，strlen特别有用，比如`char arr[] = "hello";` strlen(arr)返回的结果是5。
    * strlen不能用于不以字符终结符'\0'结尾的字符数组，会导致未定义行为。  
    ```
    char arr[5] = {'H', 'e', 'l', 'l', 'o'};
    std::cout << strlen(arr); // 这会导致未定义行为，因为没有终止符'\0'。
    ```
* 获取动态内存分配的大小
    * 使用new动态分配的内存，sizeof不能直接获取分配内存的大小，需要在分配的时候显示的记录内存大小。  
    ```
    int main() {
    int n = 10;
    int* arr = new int[n];
    // std::cout << "Size of dynamic array: "<< sizeof(arr) << " bytes"<< std::endl; // 错误：这将输出指针的大小，而不是数组的大小
    std::cout << "Size of dynamic array: " << n * sizeof(int) << " bytes"<< std::endl; // 正确：手动计算数组大小
    delete[] arr;
    return 0;}
    ```
* 空类的大小
即使是空类也会有大小（不包含任何成员函数和成员变量），sizeof的返回值至少为一个字节（如果大小为 0，多个空类对象可能会拥有相同地址，违反 C++ 的对象模型）。


## extern有什么作用？extern “C”有什么作用？
1. extern主要有两个作用：
* 声明变量的外部链接：当一个变量在一个文件中声明，当在另一个文件中定义时，可以用extern来告知编译器该变量在其他地方定义，避免编译器的编译错误。（声明变量或函数在别的文件中定义；告诉编译器这个符号在别处定义，连接时再去找）    
    ```
    // file1.cpp
    int a = 10;

    // file2.cpp
    extern int a;  // 引用 file1.cpp 中定义的 a
    ```
* extern "C"：在导出C++函数符号时，通过该操作，可以保证导出的符号为C符号而不是C++符号，这样可以更好的做兼容，比如llvm编译器导出的库，通过C符号也可以让MSCV编译器正常使用。 （确保C++代码能正确链接到C语言编写的库） 
    ```
    // C 的头文件 example.h
    void c_function();

    // C++ 文件中使用
    extern "C" {
        #include "example.h"
    }
    ```

## explicit关键字的作用？
用于修饰构造函数或者转换函数，防止编译器进行隐式类型转换
1. 应用场景
* 构造函数只带一个参数但不希望发生隐式类型转换时使用
* 提高代码安全性

class Foo {
public:
explicit Foo(int x) : value(x) {}
private:
int value;
};

void func(Foo f) {
// …
}

int main() {
Foo foo = 10; // 错误，必须使用 Foo foo(10) 或 Foo foo = Foo(10)
func(10); // 错误，必须使用 func(Foo(10))
}

* 对于那些带有**默认参数**的复杂构造函数，explicit尤为重要，能防止隐式转换

## final关键字的作用？
防止类被继承或防止虚函数被覆盖。（防止继承和虚函数重写）
1. 防止类被继承
当一个类被声明为final，这个类不能被进一步集成继承。

class Base final {
// 类的实现
};

// 下面的代码会导致编译错误
class Derived : public Base {
// 类的实现
};


2. 防止虚函数被覆盖
当一个虚函数被声明为final,这个虚函数在派生类中不能被重新定义。

class Base {
public:
virtual void doSomething() final {
// 函数实现
}
};

class Derived : public Base {
public:
// 下面的代码会导致编译错误
virtual void doSomething() override {
// 函数实现
}
};

## 三种智能指针的使用场景？
### unique_ptr
这是一种独占所有权的智能指针，意味着同一时间只能有一个unique_ptr指向一个特定的对象。但unique_ptr被销毁时，他所指向的对象也会被销毁。
使用场景：
* 但你需要确保一个对象智能被一个指针所拥有时
* 需要自动管理资源（如文件句柄和互斥锁时） 
### shared_ptr
共享所有权的智能指针，多个shared_ptr可以指向同一个对象，内部引用采用计数的方法统计指向同一个对象的指针数量，只有计数为0时，对象才会被销毁。
使用场景：
* 多个所有者之间共享对象
* 需要通过复制构造函数和赋值操作符来复制智能指针时
### weak_ptr
weak_ptr是一种不用有对象所有权的智能指针，他指向一个有shared_ptr管理的对象，用于解决shared_ptr之间的循环引用问题。
使用场景：
* 访问但不拥有
* 解决shared_ptr之间循环引用的问题
* 注意weak_ptr和shared_ptr肯定要一起搭配使用
### 扩展知识
1. 使用建议：
    * 尽量使用智能指针来管理内存，而非裸指针，因为RAII机制在很多时候会靠谱很多
    * 如果没有多个所有者共享内存的需求，建议优先使用unique_ptr管理内存，因为他相对shared_ptr更加轻量
    * 使用shared_ptr一定要注意是否会造成循环引用问题，因为这会导致内存泄漏
    * shared_ptr的引用技术是安全的，但是里面的对象不是线程安全的，这点要区分
1. 为什么unique_ptr只能移动，不可复制？
因为把拷贝构造函数和赋值运算符都标记为了delete（= delete 是 C++11 引入的特性，用于显式禁用函数的默认实现。），只能移动是因为C++11支持移动语义，可以通过move操作来转移所有权。

std::unique_ptr a(new T);
std::unique_ptr b = std::move(a); // a放弃所有权，b接管资源

2. shared_ptr的原理
核心原理就是通过引用计数机制来管理资源生命周期。
* 每个 shared_ptr 内部都维护一个 控制块（control block），控制块中包含：
* 引用计数（use_count）：有多少个 shared_ptr 正在共享这块资源。
* 弱引用计数（weak_count）：有多少个 weak_ptr 在观察这块资源。
* 资源指针：指向实际对象的原始指针。
* 删除器（可选）：负责自定义释放资源的逻辑。
3. 智能指针的缺点
主要有两个缺点：  
* 性能开销，需要额外的内存来存储控制块
* 循环引用问题（AB之间相互引用，导致两者无法释放，出现内存泄漏）

//lambda表达式
auto func = [](int a, int b) { return a + b; };

std::vector::iterator it = vec.begin();

auto it = vec.begin();

1. 配合范围for循环

for (auto val : vec) { … }

### 智能指针

### RAII lock
两种封装所lock_guard和unique_lock，都是采用了RAII的思想（在对象构造时获取资源，在对象析构时释放资源）
1. std::lock_guard<std::mutex>
简单快速，作用域结束时自动解锁

std::mutex mtx;
void func() {
std::lock_guardstd::mutex lock(mtx); // 上锁
// 临界区代码
} // 离开作用域，自动解锁

1. std::unique_lock<std::mutex>
更加灵活，支持延迟上锁，手动解锁，条件变量等

std::unique_lockstd::mutex lock(mtx);
cond_var.wait(lock); // 用于等待条件变量

1. 两种锁的区别
lock_guard 更轻便适合简单加锁场景，而 unique_lock 更灵活，适合需要条件变量或手动控制锁行为的复杂并发逻辑。
### std：：thread
std：：thraed是C++11引入的标准线程类，用于创建和管理操作系统级别的线程。
1. 常用接口
join：阻塞线程知道子线程结束
detach：让子线程独立运行，主线程无需等待
joinable：判断线程是否可被join
get_id：获取线程ID
1. 注意事项
* 线程必须join或者detach,否则进程会奔溃。
* 不能拷贝线程对象但是可以移动

std::thread t1(func);
std::thread t2 = std::move(t1); // 合法


### 左值和右值
等号左边左值，等号右边右值，可以取地址有名字的是左值，不能取地址无名字的是右值
### std：：function
std::function是C++11提供的通用可调用对象包装器，可以存储普通函数、函数指针、Lambda表达式、函数对象（重载了operator（）的类）等,使用fuction可以完全替代以前那种繁琐的函数指针设计。
`std::function<返回类型(参数类型...)> func;`
当给std::fuction填入合适的参数表和返回值后,他就变成了可以容纳所有这一类调用方式的函数封装器,同时std::fuction还可以作为回调函数,例子如下:

std::function<void(int)> f; // 这里表示function的对象f的参数是int，返回值是void
#include
#include

struct Foo {
Foo(int num) : num_(num) {}
void print_add(int i) const { std::cout << num_ + i << ‘\n’; }
int num_;
};

void print_num(int i) { std::cout << i << ‘\n’; }

struct PrintNum {
void operator()(int i) const { std::cout << i << ‘\n’; }
};

int main() {
// 存储自由函数
std::function<void(int)> f_display = print_num;
f_display(-9);

// 存储 lambda
std::function<void()> f_display_42 = { print_num(42); };
f_display_42();

// 存储到 std::bind 调用的结果
std::function<void()> f_display_31337 = std::bind(print_num, 31337);
f_display_31337();

// 存储到成员函数的调用
std::function<void(const Foo&, int)> f_add_display = &Foo::print_add;
const Foo foo(314159);
f_add_display(foo, 1);
f_add_display(314159, 1);

// 存储到数据成员访问器的调用
std::function<int(Foo const&)> f_num = &Foo::num_;
std::cout << “num_: “ << f_num(foo) << ‘\n’;

// 存储到成员函数及对象的调用
using std::placeholders::_1;
std::function<void(int)> f_add_display2 = std::bind(&Foo::print_add, foo, _1);
f_add_display2(2);

// 存储到成员函数和对象指针的调用
std::function<void(int)> f_add_display3 = std::bind(&Foo::print_add, &foo, _1);
f_add_display3(3);

// 存储到函数对象的调用
std::function<void(int)> f_display_obj = PrintNum();
f_display_obj(18);
}


### lambda表达式
`auto func = [capture] (params) opt -> ret { func_body; };
`
* capture：捕获外部变量的方式，如值捕获[=]，引用捕获[&]
* parameters:形参列表
* opt：函数选项（mutable，编译器实现时会去掉 operator() 的 const 限定，允许修改类的成员变量副本。）
* ret：返回类型（可省略，由编译器推导）
lambda表达式定义了一个匿名函数,可以捕捉一定范围的变量在函数内部使用。
1. 底层实现原理
lambda表达式本质上是一个匿名类的语法糖，编译器会将lambda表达式转变成一个带有operator()的类对象，并根据捕获方式生成相应的成员变量。
2. 如何搭配std::fuction使用
`std::function<int(int)> func = [x](int a) { return x + a; };
`
把lambda赋值给std::fuction，编译器生成了lambda类对象，std::function 接收并用类型擦除封装该类对象，使其变为多态可调用体。
